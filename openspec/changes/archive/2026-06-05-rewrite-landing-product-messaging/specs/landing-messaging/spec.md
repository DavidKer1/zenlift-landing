# landing-messaging Spec

## ADDED Requirements

### Requirement: Product-first Zenlift messaging

The landing page SHALL communicate Zenlift as a mobile workout tracker for gym users.

#### Scenario: User reads the landing page

- **WHEN** a user scans the hero and sections
- **THEN** they understand Zenlift helps them create routines, log sets, remember previous lifts, and review progress.

### Requirement: No fabricated proof

The landing page SHALL NOT use fake testimonials, fake customer logos, SaaS pricing plans, or unsupported performance claims.

#### Scenario: Template copy search runs

- **WHEN** public app files are searched for known template phrases
- **THEN** no leftover public template claims are found.
